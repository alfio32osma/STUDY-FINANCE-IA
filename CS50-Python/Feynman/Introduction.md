# Summary of the Hybrid Finance + AI Study Plan and Its Structure
## Core Objectives
* Practical Mastery (Finance + AI): Bridge the gap between theoretical knowledge and real-world application, combining financial concepts with data science and artificial intelligence.

* Portfolio & Employability: Build a robust, working portfolio on GitHub centered around a real project (a trading/market bot), preparing you for job opportunities in Málaga in the short term and internationally (such as Switzerland) in the long term.

* Real Ownership of Code: Transition from relying entirely on AI-generated code to fully understanding, testing, refactoring, and maintaining a complex 700-line Python script through a "bot reconquest" thread.

## Why It Is Structured This Way (Design Principles)
* "Learn → Integrate" Dynamic: Unlike traditional courses that accumulate endless theory before practice, every phase follows an iterative cycle: learn a specific concept using 100% free resources, practice it with isolated exercises, and immediately integrate it into your phase project and the parallel bot thread.

* Software Engineering Fundamentals First: The plan purposely schedules software engineering basics (Git, testing with pytest, and code style guidelines) before jumping into advanced machine learning, avoiding poor practices and messy code.

* Optimized for Working Professionals: Designed around realistic time constraints (~19–20 hours per week) split between continuous work shifts and longer weekend blocks, using active recall (Anki) and the Feynman Technique to maximize retention without burning out.

* Step-by-Step Bot Reconquest: Starting with a lightweight Phase 0 (reading and documenting the bot structure with AI as a tutor), the plan progressively hands over control of the code to you—moving from writing tests and database migrations to adding machine learning logic and automated reporting—ensuring you truly own your project by the end.
